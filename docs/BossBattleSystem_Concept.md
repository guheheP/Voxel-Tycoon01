# 企画書：ボスバトルシステムの追加

## 1. 概要・目的
従来の「エリアへの派遣と待機」というシステムに加え、特定エリアの最深部で「ボス」に挑む本格的なRPG風の戦闘システム（ボスバトルシステム）を追加する。
育てた冒険者と調合した自慢のアイテムを駆使して戦う明確な目標（マイルストーン）を提供することで、プレイヤーの「アイテム作成」と「冒険者育成」のモチベーションを飛躍的に高める。

## 2. コア要件
* **エリアのアンロック（進行度管理）:** 現在のエリアに設定されたボスを撃破することで、次のより難易度の高いエリア（新しい素材が入手できるエリア）が解放されるゲームループとする。
* **リトライ推奨のデザイン:** ボスバトルで敗北した場合のペナルティは設けず、何度でも無料で再挑戦可能にする。（ボスに勝てない場合は、お店に戻ってより良いアイテムを調合したり、他のエリアで冒険者を育成し直すというサイクルを生み出す）

## 3. バトルシステムの詳細
本システム最大の要である「アクティブタイムバトル（ATB）形式」の戦闘システム仕様。

* **冒険者のフルオートバトル機能:**
    * 参加する冒険者と相手ボスは、それぞれ独自の「待機ゲージ（アクションゲージ）」を持ち、一定のリアルタイム経過で蓄積していく。
    * ゲージがMAXに達したキャラクターから、順番に自動で攻撃などのアクションを実行する。冒険者の行動操作は基本的におまかせとなる。
* **プレイヤーの介入（アイテム支援）:**
    * プレイヤーの主役は「アイテム」。戦闘中、プレイヤーは好きなタイミングで倉庫内の特定のアイテムを使用できる。
    * **アイテム使用の戦術例:**
        * **回復アイテム（ポーション）:** 敵の攻撃を受けてピンチになった冒険者のHPを回復してあげる。
        * **強化アイテム（武器・霊薬）:** 冒険者に一定時間「攻撃力アップ」や「行動速度アップ（ゲージ変動）」のバフをかける。
        * **攻撃アイテム（爆弾）:** プレイヤー自身がボスに直接ダメージを与える。

## 4. UI/UX設計
* **専用のバトルビュー:**
    * ボス戦を開始すると専用のバトル画面UI（全画面オーバーレイ等）に移行する。
    * **排他処理:** バトル中は「お店の営業」や「調合」など他の作業はできず、時間の進行をバトルに集中させる。（裏の経営時間は一時停止するか、専用のものにする）
* **画面構成案:**
    * **上部/中央:** 巨大なボスの3Dモデル（または立ち絵）と、視覚的にわかりやすい長大なHPゲージ。
    * **下部（味方陣営）:** 参加中の冒険者のカード（HPゲージ、ATBゲージの進行具合）。
    * **手元（プレイヤー陣営）:** 持ち込んだ「戦闘用アイテム」のスロット。即座にクリックして使えるように配置。

## 5. 今後の実装へのステップ（技術的な影響）
1. **ステータス体系の再構築:** 冒険者やエリア敵に対する「HP・攻撃力・防御力・素早さ（ATB速度）」データのパラメーター追加。
2. **アイテムの戦闘利用システム開発:** アイテムデータに戦闘時の効果（`battleEffect`機能など）を実装。
3. **専用バトルシーンマネージャーの開発:** 通常の経営シーンとは分離された戦闘ループ（SceneManagerの拡張やバトル専用コンポーネント化）。
4. **イベントフラグとの結合:** ボスを倒したことをトリガーとし、QuestSystemなどを介して次のエリアフラグをアンロックするフロー。
